Що таке Claude Code CLI?

Агент — це не просто чатбот. Він бачить ваші файли, виконує команди та приймає рішення самостійно.

Звичайний чатбот

  • Відповідає текстом
  • Не бачить ваших файлів
  • Не виконує команди
  • Кожен раз з нуля

Claude Code CLI Агент

  • Читає та редагує файли проєкту
  • Виконує команди в терміналі
  • Пам'ятає контекст між сесіями
  • Генерує скрипти та запускає їх
  • Підтримує MCP-протокол

Як встановити та запустити агента

4 кроки від нуля до повноцінної роботи з MikroTik через AI

1

Встановити Node.js та Claude Code CLI

Claude Code — це CLI-програма, яка встановлюється через npm. Потрібен Node.js 18+.

# 1. Встановити Node.js (якщо немає) # macOS: brew install node # 2. Встановити Claude Code CLI npm install -g @anthropic-ai/claude-code # 3. Перевірити встановлення claude --version
2

Авторизуватись та запустити

Потрібен API ключ Anthropic або підписка Claude Pro/Max. Перший запуск — в робочій папці проєкту.

# Запустити в папці MikroTik-проєкту cd ~/Projects/Mikrotik_Agent claude # При першому запуску браузер відкриє сторінку авторизації # Або встановити API ключ вручну: export ANTHROPIC_API_KEY="ваш-ключ"
3

Створити CLAUDE.md — "мозок" агента

Файл CLAUDE.md описує роль агента, правила та структуру проєкту. Агент читає його при кожному запуску.

# CLAUDE.md — приклад для MikroTik проєкту ## Роль агента Ти — мережевий інженер-автоматизатор. ## Середовище - Python: venv/bin/python3 - SSH: ssh mikrotik (alias у ~/.ssh/config) - Роутер: 10.0.0.254, user: Stas ## Правила - НІКОЛИ не міняти конфіг без явної команди - ЗАВЖДИ робити backup перед змінами - Логувати всі операції у logs/
4

Налаштувати SSH до MikroTik

Агент підключається до роутера через SSH-ключ (не пароль). Це безпечно і зручно.

# ~/.ssh/config — аліас для MikroTik Host mikrotik HostName 10.0.0.254 User Stas IdentityFile ~/.ssh/id_rsa PubkeyAcceptedAlgorithms +ssh-rsa HostKeyAlgorithms +ssh-rsa # Перевірити підключення ssh mikrotik # → [Stas@StanislavZubar98] >

Порада для початківців

Не потрібно знати мережеві протоколи. Просто описуй завдання природною мовою: "Хочу закрити всі порти на WAN крім потрібних" або "Зроби щоб тільки мій MacBook міг заходити в роутер". Агент перетворить це на конкретні RouterOS команди.

CLI + WinBox: два вікна — одна стратегія

SSH / CLI
Виконання команд
  • Агент генерує RouterOS команди
  • Ти копіюєш і виконуєш у SSH
  • Результат повертаєш агенту
  • Backup через scp автоматично
WinBox
Візуальний контроль
  • Відкрий IP Firewall → Filter Rules
  • Перевір що правила є і в правильному порядку
  • Переглянь Interfaces, DHCP Leases
  • Tools → WireGuard → Peers — перевір трафік
# Приклад реального діалогу з агентом: Ти: "Перевір чи є forward rule для WireGuard клієнтів" Агент: Скинь вивід /ip firewall filter print Ти: [вставляєш вивід з SSH] Агент: Правило відсутнє. Виконай: /ip firewall filter add \ chain=forward \ src-address=10.0.99.0/24 \ action=accept \ comment="WireGuard clients to LAN" \ place-before=3 Ти: [виконуєш, відкриваєш WinBox для перевірки]
Зв'язка AI

Два комп'ютери — два AI

Практика показала: один AI не завжди достатньо. ChatGPT + Claude Code = максимальна ефективність для новачка

ChatGPT (другий ПК)

Пояснює концепції, формулює завдання, допомагає зрозуміти що відбувається та які питання ставити Claude

Claude Code CLI (основний)

Виконує технічну роботу: аналізує конфіги, генерує команди, зберігає файли, веде проєкт

Ти (оператор)

Копіюєш результати між AI, виконуєш команди на роутері, контролюєш через WinBox, приймаєш рішення

Важлива правда про AI + мережі

Налаштування обладнання — це не лінійний процес. Завжди є баги, несподівані поведінки, залежності між правилами.

AI не має фізичного доступу до роутера — він не бачить результату команди доки ти не скинеш йому вивід.

Критичні зміни — завжди backup перед виконанням і уважно читай що пропонує агент.

Головна цінність: ти навчаєшся поки робиш. Агент пояснює що і чому — це краще за будь-який курс.
